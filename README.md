# Local Legal Contract Audit Backend (Web Version)

这是一个部署在本地局域网环境中的英文合同审计 Web 服务。当前版本已经完成 Web 化迁移，支持上传 `.docx`、执行规则审计、生成带修订痕迹的修订版文档，并在浏览器中提供高保真 HTML 预览。

项目的长期目标仍然是“本地化、高精度、跨语言”的合同语义检索与智能审计系统，但**以当前代码为准**，主审计流程仍以规则驱动为主；语义检索引擎已经独立实现，但尚未完全接入上传审计主链路。

## 当前状态

- 已实现：Web 上传、请求级临时目录隔离、规则审计、红线修订、HTML 预览、下载修订版。
- 已具备但未完全接入主链路：文档切片、Embedding + Reranker 双阶段检索引擎。
- 仍在规划中：基于 `needs-260211.txt` 的完整语义审计、邮件对照审计、登录/历史、LLM 集成。

## 快速开始

### 1. 环境准备

- Python 3.11+
- 安装依赖：

```bash
pip install -r requirements.txt
```

### 2. 启动服务

```bash
python web_server.py
```

默认会监听 `0.0.0.0:8000`，终端会输出本地和局域网访问地址。

注意：服务启动时会尝试初始化语义检索模型。若本地模型不完整且当前环境无法下载，状态接口会显示初始化失败；这不会改变文档对“当前主审计流程仍以规则审计为主”的判断，但会影响后续接入语义能力。

### 3. 浏览器使用

- 上传 `.docx` 合同。
- 查看左侧修订预览和右侧审计卡片。
- 点击卡片可跳转并高亮对应位置。
- 下载带红线修订痕迹的 `.docx` 文件。

## 代码入口

- `web_server.py`：FastAPI 服务入口。
- `api_interface.py`：业务编排层。
- `core/word_processor.py`：规则审计与修订注入。
- `core/preview_generator.py`：DOCX 到 HTML 预览。
- `core/search_engine.py`：语义检索引擎。
